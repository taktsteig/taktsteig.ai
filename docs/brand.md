## 🧭 Core Philosophy

> **Not automation.  
> Guidance.**

We do not optimize for scale, speed, or volume.

We optimize for:
- **Intentional cycles**
- **Human-led AI**
- **Rising impact over time**

We believe the future of AI is not autonomous systems —  
but **guided systems**, where humans remain at the center of decision, rhythm, and responsibility.

### Core Principles:
1. **Takt before Tempo**  
   Every action follows a repeating cycle — not a pipeline.
2. **Steig through Practice**  
   Impact increases not by more AI — but by better human guidance.
3. **No Black Boxes**  
   Every tool is open, explainable, repeatable.
4. **Minimal Surface**  
   Less prompt, less interface, less noise — only what is necessary.
5. **Rhythm as Infrastructure**  
   The system is not in the code — it is in the repetition.

## 📖 Founding Story

`taktsteig` began in 2026 not as a company, but as a question:

> “Why do AI systems feel faster — but leave us more lost?”

After years of building and using automated workflows, the founder noticed a pattern:  
- Systems *produced more* — but *decisions weakened*.  
- Outputs grew — but responsibility faded.  
- People followed the tool — instead of leading it.

A reversal was needed.

Not toward "more control" — but toward **clear cycles**, **intentional rhythm**, and **rising competence**.

`taktsteig` was born from one insight:  
> **If the system repeats, the human must rise — not disappear.**

It started with a notebook.  
Then a script.  
Then a loop.  
Now: an open path.

## 🎯 Ethical Standing

We are not neutral. We take a stand.

### We believe:
- AI must not erase ownership of decisions.
- Users must always be able to answer: *"Why did this happen?"*
- Systems should support discipline — not replace judgment.

### We reject:
- Fully autonomous agents that act without review
- Interfaces that hide process behind prompt chains
- Tools that grow smarter while users grow passive

### Our commitments:
- **No telemetry** — no tracking, no analytics, no user data collection
- **No SaaS lock-in** — everything runs locally or in your environment
- **Open source first** — all core tools published under permissive licenses
- **No venture pressure** — funded by work, used by practice

We are not building for growth.  
We are building for **stewardship**.

## ⚙️ CI/CD & Technical Practice

We deploy not for velocity — but for **rhythmic iteration**.

### Workflow: The Cycle
1. **Draft** — change made locally
2. **Commit** — labeled with `cycle/`, `takt/`, or `steig/`
3. **Test** — only static checks (shellcheck, markdown-lint)
4. **Deploy** — GitHub Pages on push to `main`
5. **Review** — human checks: *Did impact rise?*

### Infrastructure
- **Repo**: `taktsteig/taktsteig.ai` → GitHub Pages  
  → Source of truth
- **Tools**: Simple shell scripts, published via GitHub releases
- **No cloud backend** — all tools run locally
- **No database** — state lives in plain text, versioned

### Deployment
- On every `git push` to `main`, GitHub Actions:
  - Runs lint
  - Rebuilds site
  - Deploys `taktsteig.ai` in <10s
- Manual `curl`-installable scripts via `/bin/`
- No CI gates beyond: *Does it run? Is it documented?*

> We do not automate release.  
> We **schedule** it.
